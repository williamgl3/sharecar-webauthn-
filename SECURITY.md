# Seguridad

Si encuentras una vulnerabilidad de seguridad, por favor informa de forma
privada a los mantenedores. No publiques exploits en issues públicos.

Pasos recomendados:
1. No intentes explotar la vulnerabilidad en entornos de terceros.
2. Reúne información mínima reproducible (pasos, entorno, versiones).
3. Envía un correo a los mantenedores o abre un issue marcado como privado.

Buenas prácticas para el repositorio:
- Nunca comitear claves privadas ni secrets.
- Use variables de entorno y servicios de secretos (Netlify env vars,
  Azure Key Vault, GitHub Secrets) para almacenar credenciales.

Para preguntas o informes, cree un issue privado o contacte por el canal del
proyecto.
